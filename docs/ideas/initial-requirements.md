# Badminton Event Hub - 現行機能一覧

## 1. 認証・ユーザー管理

- メール/パスワードによるサインアップ・ログイン
- Google OAuth サインイン
- メール認証(メール登録時)
- 初回ログイン時にプロフィール設定画面へ誘導
- 表示名の設定・編集(2〜20文字)

## 2. イベント(練習会)管理

### 作成
- 入力項目: タイトル、開始日時、終了日時、会場名、住所、市区町村(宮城県内38箇所)
- レベル選択: 初心者 / 初級 / 中級 / 上級 / すべてのレベル(補足メモ任意)
- 定員(ビジター枠)、参加費
- 説明文、参加ルール、持ち物・装備、備考
- 募集締切(任意): イベント開始の1〜72時間前から選択

### 編集
- 全項目の編集が可能
- イベント開始後は編集不可
- 主催者のみ編集可

### コピー
- 既存イベントの情報をコピーして新規作成(日時以外をプリフィル)

### キャンセル
- 主催者がイベントをキャンセル可能
- 承認済み参加者全員にメール通知

## 3. イベント検索・閲覧

- トップページにイベント一覧をカード表示
- フィルター: 日付(今日/今週/来週/今月/来月/カスタム)、市区町村、レベル
- イベント詳細ページ: 全情報の表示、残り枠数、主催者名
- 申請ボタン(状態に応じて表示が変化: 申請可/保留中/承認済み/ブロック済み等)
- SEO対応: JSON-LD構造化データ、OGメタタグ

## 4. 申請管理

### 参加者側
- イベントへの申請(任意コメント付き)
- 申請のキャンセル(保留中・承認済みの申請)

### 主催者側
- 全申請の一覧表示(保留中/承認済み/拒否・キャンセル)
- 申請の承認(メール通知付き)
- 申請の拒否(メール通知付き)
- 拒否と同時にユーザーブロック

### 申請の制約
- 公開中のイベントのみ申請可
- 募集締切前のみ申請可
- ブロックされていないこと
- 既存の保留中/承認済み申請がないこと
- 残り枠があること
- 自分のイベントには申請不可

## 5. チャット機能

- 承認済み参加者と主催者間の1対1チャット
- リアルタイムメッセージ配信
- メッセージ上限500文字
- クイックメッセージテンプレート(「迷っています」「遅れます」「体調不良でキャンセル」等)
- 既読状態のトラッキング
- イベント終了後48時間でチャット期限切れ
- 新規メッセージ受信時にメール通知

## 6. ダッシュボード(主催者向け)

- 今後のイベント一覧(ステータスバッジ、参加者数/定員、クイックアクション)
- 過去のイベント一覧(直近5件を折りたたみ表示)
- 保留中の申請数バッジ
- イベントがない場合の作成誘導

## 7. マイページ(参加者向け)

- 申請一覧をステータス別に表示(保留中/承認済み(今後)/承認済み(過去)/拒否/キャンセル)
- イベントごとのアクション(詳細表示、主催者にメッセージ、参加キャンセル)
- プロフィール編集リンク

## 8. 通知

### アプリ内通知
- ヘッダーに保留中の申請数バッジ(主催者向け)
- ヘッダーに未読チャット数バッジ(全ユーザー向け)

### メール通知
- 新規申請 → 主催者へ
- 申請承認 → 申請者へ
- 申請拒否 → 申請者へ
- イベントキャンセル → 承認済み参加者全員へ
- 参加者キャンセル → 主催者へ
- 新規チャットメッセージ → チャット相手へ

## 9. ブロック機能

- 主催者がユーザーをブロック可能
- ブロックされたユーザーはその主催者の全イベントへの申請が不可
- チャットアクセスも制限

## 10. その他

- レスポンシブデザイン(モバイルファースト)
- ページ遷移時のローディング表示
- 日時はすべてJST表示
